# 数据分析文件说明

## analytics_data.json 字段说明

本文件包含从B站UP主空间爬取的视频数据，经过处理后提取出的关键分析维度数据。

### 视频基本信息字段

| 字段名 | 说明 |
|--------|------|
| title | 视频标题，描述视频内容的简短文字 |
| bvid | BV号，B站视频的唯一标识符 |
| typeid | 分区ID，表示视频所属的内容分类（如游戏、音乐、动画等） |
| created | 创建时间，以Unix时间戳格式表示的视频发布日期 |
| length | 视频时长，格式为"分钟:秒钟" |

### 视频互动数据字段

| 字段名 | 说明 |
|--------|------|
| play | 播放量，视频被观看的次数 |
| comment | 评论数，观众对视频发表的评论数量 |

### 高级互动数据字段

这些字段主要出现在系列视频中，普通视频这些字段值通常为0：

| 字段名 | 说明 |
|--------|------|
| like | 点赞数，观众对视频表达喜欢的数量 |
| coin | 投币数，观众对视频投币的数量（B站特有的支持方式） |
| favorite | 收藏数，被用户收藏的次数 |
| danmaku | 弹幕数，视频中发送的弹幕数量 |
| share | 分享数，视频被分享的次数 |

## 数据说明

1. 只有包含 `meta` 信息的系列视频才具有完整的互动数据（like, coin, favorite, danmaku, share）
2. 普通视频的高级互动数据字段值都为0，因为没有通过 `meta.stat` 提供这些详细统计数据
3. 分区ID对应关系可以在 `user_videos_data.json` 文件中的 `tlist` 字段中找到

## 数据问题说明

在实际的B站视频中，所有视频都应该具有点赞、投币、收藏等数据，但是在当前收集的数据中，大部分普通视频的这些字段显示为0。这是因为：

1. **API限制**：通过 `user.get_videos()` 方法只能获取部分统计数据
2. **数据层级问题**：普通视频的详细统计数据存储在不同的API中

## 解决方案

为了解决这个问题，我们提供了额外的脚本 `detailed_video_stats.py` 来获取每个视频的完整统计数据：

1. 运行原始爬虫获取基本数据：
   ```
   python user_videos_crawler.py
   ```

2. 运行详细数据收集脚本获取完整统计数据：
   ```
   python detailed_video_stats.py
   ```

3. 脚本会生成 `analytics_data_detailed.json` 文件，其中包含所有视频的完整统计数据

## 使用建议

这些数据可以用于以下分析：
- 不同类型视频的表现对比
- 视频时长与播放量的关系
- 发布时间对视频表现的影响
- UP主内容策略的分析
